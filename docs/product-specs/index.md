# Product Specs

## Problem
Manually populating 7+ markdown files, updating brand tokens, creating Issues, and setting up AGENTS.md for every new project scaffold. Tedious and inconsistent.

## Solution
A `project-config.yaml` drives an automated scaffold population script. The frontend tool generates the config visually.

## Requirements
- **Project Info:** Name, repo name, description, domain, GitHub owner
- **Brand:** 7 color pickers, font selectors
- **Pages:** Dynamic page list with section chips, reorderable
- **Tech Stack:** Framework, deployment, SSR, CMS, agent assignment
- **Export:** YAML (frontmatter), JSON, Markdown

## Pages Generated by Config
| Page | Route | Sections |
|------|-------|----------|
| Home | `/` | hero, recent-configs, cta |
| New | `/new` | project-info, brand, pages, tech, deploy |
| Review | `/review` | config-preview, export |
